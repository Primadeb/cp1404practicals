### Main Lessons Learned about Clean Code

-Consistent Naming: Using descriptive and consistent names for variables, functions, and classes 
improves readability and understanding.

-Proper Commenting: Adding comments to explain complex logic or clarify the 
purpose of code sections helps other developers (and your future self) understand the code.

-Modularization: Breaking down code into smaller, reusable modules or functions makes it easier
to maintain, debug, and test.
